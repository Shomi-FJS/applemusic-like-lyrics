## TTML-歌词源

### 数据来源

- **主要来源**: [Steve-XMH/amll-ttml-db](https://github.com/Steve-XMH/amll-ttml-db) GitHub 仓库 (`/ncm-lyrics` 目录)
- **回退来源**: 通过 `EXTERNAL_NCM_API_URL` 环境变量配置的外部 API

### 安装与运行

#### 安装依赖

推荐使用 pnpm：

```bash
pnpm install
```

或使用 npm：

```bash
npm install
```

或使用 yarn：

```bash
yarn install
```

#### 配置环境变量

**开发环境**: 创建 `.env` 文件在项目根目录：

```env
EXTERNAL_NCM_API_URL=https://NeteaseCloudMusicApi/lyric/new
PORT=3000
```

> **重要**: 将 `.env` 文件添加到 `.gitignore`

**生产环境**: 通过部署平台或系统设置相应的环境变量

#### 开发模式（带热重载）

```bash
d: & cd D:\a\applemusic-like-lyrics\Lyric-Atlas-API & pnpm run dev
```

服务器将在 `http://localhost:3000`（或 `PORT` 环境变量指定的端口）启动

#### 构建生产版本

```bash
pnpm run build
```

确保 `tsconfig.json` 中 `outDir` 配置为 `./dist` 并且 `noEmit` 为 `false` 或已移除

#### 运行生产版本

```bash
pnpm run start
```

## WebSocket 架构流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        外部程序（客户端）                         │
│  发送切换歌曲指令:                                                │
│  V1: Binary(0x000E)  或  V2: {"type":"command","command":"ForwardSong"} │
└──────────────────────┬──────────────────────────────────────────┘
                        │ WebSocket消息
                        ▼
┌─────────────────────────────────────────────────────────────────┐
│              WebSocket服务器 (server.rs)                         │
│  1. 接收消息                                                      │
│  2. 根据协议类型处理:                                             │
│     - V1: parse_body() → 转换为v2::Payload                       │
│     - V2: 直接解析为v2::Payload                                  │
│  3. 通过Channel发送到Tauri后端                                    │
└──────────────────────┬──────────────────────────────────────────┘
                        │ Channel消息
                        ▼
┌─────────────────────────────────────────────────────────────────┐
│              Tauri后端 (lib.rs + player.rs)                     │
│  1. 接收Channel消息                                              │
│  2. 转发给播放器核心: handler.send(msg)                          │
└──────────────────────┬──────────────────────────────────────────┘
                        │ AudioThreadEventMessage
                        ▼
┌─────────────────────────────────────────────────────────────────┐
│              播放器核心 (player-core)                            │
│  1. process_message() 处理消息                                   │
│  2. 执行切换歌曲:                                                │
│     - 更新 current_play_index                                    │
│     - 获取新歌曲信息                                              │
│     - start_playing_song() 开始播放                              │
│  3. sync_ui() 同步状态                                           │
└──────────────────────┬──────────────────────────────────────────┘
                        │ AudioThreadEvent
                        ▼
┌─────────────────────────────────────────────────────────────────┐
│              前端UI更新                                          │
│  接收事件并更新UI显示                                             │
└──────────────────────┬──────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────────┐
│              WebSocket服务器广播状态                             │
│  broadcast_payload() 向所有连接的客户端广播:                     │
│  - 新歌曲信息 (SetMusic)                                         │
│  - 封面 (SetCover)                                               │
│  - 歌词 (SetLyric)                                               │
│  - 播放状态 (Paused/Resumed)                                     │
└──────────────────────┬──────────────────────────────────────────┘
                        │ WebSocket消息
                        ▼
┌─────────────────────────────────────────────────────────────────┐
│              外部程序（客户端）                                  │
│  接收状态更新并同步显示                                           │
└─────────────────────────────────────────────────────────────────┘
```
